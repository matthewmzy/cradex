# CRA: Cascaded Residual Adaptation for Dexterous Manipulation

Axis-decomposed multi-stage residual policy learning for robust in-hand manipulation.

## Core Idea

Standard domain randomization (DR) trains a single policy to handle **all** physical parameter variations simultaneously. CRA decomposes this into sequential stages, each learning to adapt to **one parameter axis** through a dedicated adaptation module:

```
Stage 0:  Base policy π₀ (fixed parameters)
Stage 1:  + gravity adaptation    (freeze π₀, train A₁+ρ₁)
Stage 2:  + mass adaptation       (freeze all above, train A₂+ρ₂)
Stage 3:  + friction adaptation   (freeze all above, train A₃+ρ₃)
...
```

**Final action (all residuals computed in parallel):**
```
a = clip( π₀(o) + Σᵢ ρᵢ(o, Aᵢ(history)),  a_min, a_max )
```

Each adaptation encoder `Aᵢ` is a GRU that maps recent observation-action history to a compact latent vector `zᵢ`. Each residual head `ρᵢ` maps `(obs, zᵢ)` to an action correction. Because the structure is additive, all modules run **in parallel** at inference time.

### Why not just full DR?

| | Full DR | CRA |
|---|---------|-----|
| Each module learns | Everything at once | One axis only |
| Unseen param combos | OOD → failure | Compositional generalization |
| Interpretability | Black box | Per-axis latent analysis |
| Inference | Single network | Parallel additive residuals |

## Installation

### 1. System requirements

- Ubuntu 20.04+ with NVIDIA GPU (RTX 3070+ recommended)
- CUDA 11.7+
- Python 3.10+

### 2. Install IsaacGym

IsaacGym Preview 4 is required for the simulation environment. Download from NVIDIA:

```bash
# Download from: https://developer.nvidia.com/isaac-gym
# (requires NVIDIA Developer account, free registration)

# After downloading:
tar xzf IsaacGym_Preview_4_Package.tar.gz
cd isaacgym/python
pip install -e .

# Verify installation:
python -c "from isaacgym import gymapi; print('IsaacGym OK')"
```

### 3. Install CRA

```bash
cd cra_dexterous
pip install -e ".[analysis]"
```

### 4. Download object assets

```bash
# Download YCB objects for multi-geometry experiments
python scripts/download_assets.py --dataset ycb

# Or download all asset info
python scripts/download_assets.py --dataset all
```

## Usage

### Train CRA policy

```bash
# Full CRA pipeline (base + 4 adaptation stages)
python scripts/train.py \
    --method cra \
    --config configs/method/cra.yaml \
    --task-config configs/task/in_hand_rotation.yaml \
    --num-envs 4096 \
    --seed 42 \
    --experiment-name cra_rotation

# Custom axis order and fewer stages
# Edit configs/method/cra.yaml:
#   axis_order: ["friction", "object_mass"]
```

### Train baselines

```bash
# Full DR (all randomization from the start)
python scripts/train.py \
    --method full_dr \
    --config configs/method/full_dr.yaml \
    --task-config configs/task/in_hand_rotation.yaml \
    --experiment-name fulldr_rotation

# RMA (single adaptation module)
python scripts/train.py \
    --method rma \
    --config configs/method/rma.yaml \
    --task-config configs/task/in_hand_rotation.yaml \
    --experiment-name rma_rotation
```

### Evaluate

```bash
python scripts/eval.py \
    --checkpoint outputs/cra_rotation/checkpoints/stage_4_friction.pt \
    --method cra \
    --num-episodes 200 \
    --num-envs 256
```

### Analyze adaptation modules

```bash
# t-SNE of latents + linear probes + ablation
python scripts/visualize_adaptation.py \
    --checkpoint outputs/cra_rotation/checkpoints/stage_4_friction.pt \
    --num-steps 500 \
    --output-dir outputs/cra_rotation/analysis \
    --run-ablation
```

Outputs:
- `tsne_{axis}.png` — t-SNE of each stage's latent, colored by DR param
- `linear_probe_results.json` — R² scores for predicting DR params from latents
- `ablation_results.json` — performance with each stage disabled
- `ablation_plot.png` — bar chart of ablation results

### Monitor training

```bash
tensorboard --logdir outputs/
```

## Key Design Decisions

### Axis decomposition order

We recommend ordering by **single-axis reward drop** (most impactful first):

1. **Gravity direction** — largest effect on manipulation dynamics
2. **Gravity magnitude** — changes effective weight
3. **Object mass** — affects inertia and required forces
4. **Friction** — changes grasp stability

The first axis gets the simplest learning context (all other params fixed). The last axis must adapt in the presence of all other variations.

### Architecture hyperparameters

| Component | Default | Notes |
|-----------|---------|-------|
| Base actor MLP | [512, 256, 128] | Standard for dexterous manipulation |
| Adaptation encoder | 2-layer GRU, hidden=128, latent=16 | 16D latent is sufficient per axis |
| Residual head | [256, 128] | Small; init near zero |
| History window | 50 steps (1s at 50Hz) | Enough for system identification |

### Residual initialization

Each residual head is initialized with near-zero output weights (scale=0.01). This ensures that adding a new stage does not disrupt the existing policy — the initial residual contribution is negligible, and the new module gradually learns to produce meaningful corrections.
